Grid layouts are the difference between a website that looks designed and one that merely *functions*. Tailwind's grid utilities make complex, asymmetric compositions achievable without writing a single line of custom CSS — if you know the patterns.

## Why Grid Over Flexbox?

Flexbox is excellent at one-dimensional layouts: a row of nav links, a stack of cards. Grid excels at *two-dimensional* layouts: something that controls both row and column simultaneously.

The key insight:

- Use **Flexbox** when content drives the size
- Use **Grid** when the *layout* drives the size

For editorial-quality page design, you almost always want Grid.

## The 12-Column Foundation

The 12-column grid is the newspaper's gift to the web. Twelve is divisible by 2, 3, 4, and 6 — which gives you half, thirds, quarters, and sixths without fractions.

A standard bento layout:

- **8-column main content** + **4-column sidebar** = classic editorial two-column
- **4 + 4 + 4** = three equal columns for cards
- **6 + 3 + 3** = featured card with two smaller companions

## The Bento Breakthrough

The "bento grid" — named for the Japanese lunchbox — uses `row-span` to create asymmetric, visually interesting layouts where a featured item takes up more vertical space than its siblings.

> "Asymmetry communicates hierarchy. The eye goes to the largest thing first. Design that direction intentionally."

The key classes:

- `col-span-{n}` — how many columns an item spans
- `row-span-{n}` — how many rows an item spans
- `grid-rows-{n}` — define explicit row tracks on the container

## Responsive Considerations

A bento that works beautifully at 1280px can collapse horribly on mobile. The pattern to follow:

- **Mobile**: single column, stacked, largest item first
- **Tablet**: 2-column, drop `row-span` complexity
- **Desktop**: full bento asymmetry

This means building mobile-first and layering grid complexity with `md:` and `lg:` prefixes.

## The Gap System

Tailwind's `gap-{n}` classes control the gutters between cells. For neobrutalist layouts:

- Use **consistent gaps** — they create a disciplined grid rhythm
- **Never** use negative margins to fake grid overlap — use `z-index` and absolute positioning instead
- For bento-style designs, a `gap-4` (16px) gutter creates density; `gap-6` (24px) creates air

## What Kills a Grid Layout

The most common mistakes:

- Using `auto` columns when explicit columns are needed
- Mixing grid and flex in ways that fight each other
- Forgetting `min-h-0` on grid children that contain overflow content
- Setting fixed heights instead of letting content define row height

The grid is a *constraint*. Constraints are freedom. Embrace the discipline and the compositions will follow.
